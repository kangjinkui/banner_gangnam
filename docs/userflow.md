# 최종 유저플로우 설계 문서

본 문서는 정당별 현수막 관리 시스템의 유저플로우를 종합한 최종 완성본이다.  
구성: 입력 → 처리 → 출력, 엣지케이스 대응 포함.

---

## 유저플로우 #1 — 현수막 등록(Create Banner)

### 입력
- 이미지 파일 선택
- 주소 텍스트 입력
- 정당 선택(드롭다운)
- 문구 입력
- 시작일/종료일 선택
- 메모 입력
- 제출 트리거

### 처리
1. 클라이언트 유효성 검증 (필수값, 길이, 날짜 규칙, 파일 규칙)
2. 주소 → 좌표 변환 (Kakao Geocoding)
3. 중복/이상치 점검 (좌표/기간 중복 여부)
4. 이미지 업로드 (Supabase Storage)
5. API 요청(POST /api/banners)
6. 서버측 검증 (권한, 참조 무결성, 비즈니스 규칙, 트랜잭션)
7. 후처리 (썸네일 생성, 캐시 무효화, 감사 로그)

### 출력
- 성공: 신규 마커 표시, 목록 갱신
- 실패: 오류 메시지 및 상태 유지

### 엣지케이스
- 지오코딩 실패/다건 → 좌표 직접 입력 유도
- 파일 오류 → 업로드 차단
- 네트워크 불안정 → 멱등 키, 재시도 큐
- 트랜잭션 불일치 → 이미지 롤백 삭제
- 중복 항목 → 사용자 확인

---

## 유저플로우 #2 — 정당 관리(CRUD)

### A. 등록(Create)
- 입력: 정당명, 색상 코드, 아이콘(URL/업로드)
- 처리: 중복 검사, 아이콘 업로드, API 요청
- 출력: 목록 갱신, 신규 항목 표시
- 엣지케이스: 명칭 충돌, 파일 오류, 권한 없음

### B. 수정(Update)
- 입력: 대상 선택, 수정 필드 입력
- 처리: 중복 검사, 아이콘 교체, DB 업데이트, 캐시 무효화
- 출력: UI 갱신
- 엣지케이스: 동시 수정 충돌, 색상 오류

### C. 삭제(Delete)
- 입력: 대상 선택, 삭제 트리거
- 처리: 참조 배너 확인 후 삭제/비활성 처리
- 출력: 목록 반영
- 엣지케이스: 참조 충돌, ID 불일치

### D. 목록 조회(Read)
- 입력: 필터, 정렬, 페이지네이션
- 처리: 서버 쿼리, 페이지 응답
- 출력: 표 렌더링, 액션 버튼 노출
- 엣지케이스: 결과 없음, 대량 데이터

---

## 유저플로우 #3 — 지도 표시(Map View)

### 입력
- 지도 진입, 이동/확대/축소
- 필터 선택(정당, 행정동, 기간)
- 마커 클릭

### 처리
1. Kakao Map 초기화
2. Supabase에서 배너 데이터 조회
3. 정당 메타 로딩
4. 마커/클러스터 표시 (정당별 스타일, 기간 상태 반영)
5. 팝업 상세 표시
6. 상호작용 이벤트 처리 (이동/확대 → 재쿼리)
7. 성능 최적화 (클러스터링, 가상화 렌더링)
8. 오류/권한 처리

### 출력
- 지도에 마커 및 팝업 표시
- 필터/검색 반영된 결과
- 동기화된 목록/범례

### 엣지케이스
- 데이터 과다 → 클러스터링 강제
- 아이콘 누락 → 기본 아이콘 대체
- 좌표 이상치 → 제외 처리
- 팝업 과다 열림 → 단일 인스턴스 유지
- 오프라인 → 캐시 데이터 표시

---

## 유저플로우 #4 — 관리자 목록(Admin List)

### 입력
- 필터 선택(정당, 행정동, 기간)
- 정렬, 페이지 이동
- CSV/Excel 다운로드
- 행 액션(수정/삭제)

### 처리
1. API 요청 → 서버 필터 적용
2. 목록 표 렌더링 (위치, 정당, 문구, 기간, 썸네일)
3. 기간 상태 계산(예정/진행/만료)
4. 다운로드 요청 처리 (파일 생성 후 URL 반환)
5. 행 액션 처리 (수정/삭제)

### 출력
- 목록 및 상태 하이라이트 표시
- 다운로드 파일 저장
- 수정/삭제 결과 즉시 반영

### 엣지케이스
- 결과 없음 → 안내 메시지
- 대량 다운로드 → 비동기 처리
- 썸네일 누락 → 기본 이미지
- 권한 없음 → 접근 차단

---

## 유저플로우 #5 — 권한 및 보안(Security & Roles)

### 입력
- 로그인/세션 토큰 제출
- 기능 진입 시도

### 처리
1. 인증 (토큰 검증)
2. 권한 판별 (admin/user)
3. 권한 분기
   - 관리자: CRUD, 다운로드 가능
   - 일반: 조회 전용
4. 보안 강화 (CSRF, XSS, 파일 검증, SQL 인젝션 방지)
5. 감사 로그 기록

### 출력
- 권한 충족 시 기능 실행
- 권한 부족 시 접근 거부
- 세션 만료 시 로그인 유도

### 엣지케이스
- 토큰 만료 → 자동 재로그인
- 세션 탈취 시도 → 세션 무효화
- 권한 변경 후 세션 지속 → 서버 즉시 반영
- 오프라인 → 일반 뷰만 캐시 표시

---

# 종합 정리

위 유저플로우는 **현수막 등록 → 정당 관리 → 지도 표시 → 관리자 목록 → 권한 및 보안** 순으로 전체 기능을 커버한다.  
각 단계는 입력-처리-출력의 구조를 따르며, 예상 가능한 엣지케이스를 대응할 수 있도록 설계하였다.
